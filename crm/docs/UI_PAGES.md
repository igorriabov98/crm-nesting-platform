# Архитектура и описание UI страниц

В этом документе содержится перечень всех страниц Web-интерфейса Next.js приложения. Описание покрывает назначение, доступы, визуальные компоненты, источник данных и действия на страницах.

---

## 🟢 Аутентификация и Навигация

### `/login` — Страница входа
- **Доступ:** Публичный.
- **Компоненты:** Форма ввода (Email, Пароль), кнопка "Войти", логотип завода.
- **Данные:** `supabase.auth.signInWithPassword`.
- **Действия:** Авторизация. 
- **Особенности:** После успешного входа происходит автоматический редирект на `/dashboard` с чтением профиля для определения роли.

### `/dashboard` — Главная страница
- **Доступ:** Все аутентифицированные роли.
- **Компоненты:** Приветственный блок с метриками (карточки KPI).
- **Данные:** Агрегированные метрики в зависимости от роли (напр., кол-во активных машин, неоплаченные инвойсы).
- **Действия:** Перенаправление-роутинг (hub) в нужные департаменты. Sales-ам предлагает Sales Plan, снабжению — Supply Dashboard.

## 🔵 Раздел Администратора

### `/admin/users` — Управление пользователями
- **Доступ:** Только `planning_director`.
- **Компоненты:** Таблица сотрудников (Имя, Email, Роль, Статус `is_active`). Кнопка "Добавить".
- **Данные:** `supabase.from('users').select()`.
- **Действия:** Блокировка/Смена роли.
- **Особенности:** Встроенный поиск и фильтрация по ролям.

### `/admin/users/new` — Создание пользователя
- **Доступ:** Только `planning_director`.
- **Компоненты:** Форма с выбором Role (Enum-select) и Factory (Select).
- **Данные:** API вызов Edge Function для регистрации в auth и INSERT в `users`.
- **Особенности:** Генерирует или предлагает задать пароль сотруднику.

## 🔴 Раздел Sales Plan

### `/sales-plan` — Список машин
- **Доступ:** Все (с разным уровнем контроля).
- **Компоненты:** Data Table с основными атрибутами (Название, Тоннаж, Статус).
- **Данные:** Запрос к `machines` с агрегацией инвойсов.
- **Действия:** Директора и Sales могут нажать "Удалить".
- **Особенности:** Постраничная пагинация. Панель фильтров: по статусу, датам.

### `/sales-plan/new` — Создание машины
- **Доступ:** `sales_manager`, Директора.
- **Компоненты:** ШАД-форма (React Hook Form + Zod валидация). Поля: Название, Тоннаж, Товар, Списки чертежей. Выбор покрытия покрытия (Радио кнопки).
- **Данные:** `INSERT INTO machines`, что скрытно инициализирует 7 этапов производства.

### `/sales-plan/[id]` — Карточка машины
- **Доступ:** Все.
- **Компоненты:** Страница с Tab-навигацией: "Общая сводка", "Производство", "Снабжение", "Инвойс".
- **Данные:** Запросы `machines.single()`, `production_stages`, `supply_items`.
- **Действия:** Инженер и Технолог работают с данными снабжения внутри этой карточки.

## 🟣 Раздел Производства (Гант)

### `/production` — Таблица производства
- **Доступ:** Все.
- **Компоненты:** Детальный табличный вид (Grid) со всеми 7 этапами для всех активных машин.
- **Данные:** `production_stages` JOIN `machines`.
- **Действия:** `production_manager` может дважды кликать на ячейку этапа, чтобы быстро поменять его статус и даты в модальном окне или прямо в таблице.

### `/production/gantt` — Гант-график
- **Доступ:** Все.
- **Компоненты:** Горизонтальный таймлайн интерфейс (dnd-kit или react-calendar-timeline). 
  - Боковина: список машин. 
  - Основная зона: Цветные полоски этапов.
- **Данные:** Вычисляемый view `production_stages_with_delay`.
- **Фильтры:** По цеху (1 / 2), По этапу (сборка, малярка), По дате (DateRangePicker).
- **Цвета этапов на Ганте:**
  - Заготовка: Синий
  - Сборка: Зелёный
  - Зачистка: Жёлтый
  - Цинк: Серый
  - Малярка: Оранжевый
  - Упаковка: Фиолетовый
  - Отгрузка: Красный
  - Ночная смена: Тёмно-синий оверлей (штриховка)
  - Пропущенный этап: Прозрачный штрих
  - Просрочка: Ярко-красная обводка блока
  - Линии снабжения: Вертикальные пунктирные линии (когда придут материалы).

## 🟡 Раздел Снабжения

### `/supply` — Дашборд снабжения
- **Доступ:** Все (но снабженцы - основная аудитория).
- **Компоненты:** Таблица открытых потребностей. Группировка по `machine_id` с аккордеонами (раскрывающиеся списки). Аналитика (процент заказанного).
- **Данные:** View `supply_items_with_overdue`.
- **Действия:** Редактирование поставщика, цены и статуса IN-LINE (прямо в таблице).

### `/supply/[machineId]` — Позиции снабжения по машине
- **Доступ:** Все (Редакторы: Технолог, Инженер, Снабженец).
- **Компоненты:** Спецификация к конкретному товару. Кнопка "Добавить деталь". Поле "Инженерное подтверждение" (Checkbox).
- **Данные:** `supply_items` с `eq('machine_id', id)`.
- **Особенности:** RLS и триггеры на бэкенде защищают от редактирования "чужих" ячеек, на фронте ячейки просто disable'ятся по условию `$store.user.role`.

## 💳 Раздел Инвойсов

### `/invoices` — Список финансов
- **Доступ:** Директора, `sales_manager`.
- **Компоненты:** Финансовая сводка. Сумма оплаченных / просроченных. Дата-таблица счетов с цветовыми бейджами: Paid (Зеленый), Not Paid (Желтый), Overdue (Красный).
- **Данные:** Таблица `invoices`.
- **Действия:** Смена статуса (Dropdown).

---

## 🔔 Центр Уведомлений

### `/notifications` — Все уведомления
- **Доступ:** Все пользователи (в шапке доступен Dropdown-Popover).
- **Компоненты:** Центр сообщений (Inbox вид). Список прочитанных/непрочитанных. Кнопка "Пометить все как прочитанные".
- **Данные:** `notifications` отфильтрованные по текущему юзеру. 
- **Особенности:** Подключен Supabase Realtime (WebSocket), чтобы новые алерты "всплывали" тостом в углу экрана, и счетчик "колокольчика" увеличивался автоматически.
## Обновление 27.04.2026: страницы Parts 1-3
- Header содержит глобальный FactoryFilter: все заводы, конкретный завод, без завода. Для `production_manager` фильтр скрыт.
- `/sales-plan`: таблица показывает статус машины, завод, тип материала, товары, вес, стоимость, производство, снабжение, инвойс, автора и дату.
- `/sales-plan/[id]`: карточка показывает статус, завод, материал, totals и кнопку назначения завода для директоров, если завод не назначен.
- `/production`: показывает только машины с назначенным заводом; `production_manager` видит только свой завод.
- `/production/gantt`: два таба по заводам для глобальных ролей, один завод для `production_manager`; машины без завода исключены.
- `/supply`: применяет глобальный заводской фильтр и показывает отдельную секцию машин без назначенного завода.
- `/invoices`: применяет глобальный заводской фильтр, включая режим `no_factory`.
- `/meetings`, `/meetings/new`, `/meetings/[id]`: модуль собраний с повесткой, решениями, участниками и итоговыми задачами.
- `/dashboard`: содержит карточку "Без завода" и блок ближайшего собрания.
