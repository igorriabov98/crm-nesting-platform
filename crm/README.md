# CRM Завода

CRM система для управления производством металлоконструкций, снабжением и финансами. Реализована с использованием Next.js 14 и мощных возможностей Supabase (Auth, RLS, Edge Functions, Realtime).

## Стек
- Next.js 14, TypeScript (Turbopack), Tailwind CSS, shadcn/ui.
- Supabase (Auth, PostgreSQL, Realtime, RPC, Edge Functions).

## Установка и запуск

### 1. Клонировать репозиторий
```bash
git clone <repository_url>
cd crm-factory
```

### 2. Установить зависимости
```bash
npm install
```

### 3. Настроить Supabase
1. Создайте проект на [Supabase](https://supabase.com).
2. Скопируйте ключи подключения.
3. Создайте файл `.env.local` в корне проекта (смотрите `.env.example`, если есть) и пропишите:
```env
NEXT_PUBLIC_SUPABASE_URL=ваш_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=ваш_anon_key
SUPABASE_SERVICE_ROLE_KEY=ваш_service_key_для_cron
```
> Примечание: При сборке приложение осуществляет проверку через `validateEnv()` и может выкинуть ошибку, если ключей нет.

### 4. Применить миграции к базе данных
Выполните SQL файлы из папки `/supabase/migrations/` в порядке нумерации с `01_...` до `10_...` в вашем рабочем пространстве Supabase SQL Editor.

### 5. Создать первого пользователя
Для авторизации в системе вам нужно:
1. Зайти во вкладку **Authentication** в панели Supabase и добавить пользователя вручную (Add User). 
2. Скопировать UUID нового юзера (поле UID).
3. Перейти в SQL Editor и переопределить роль пользователя (например):
```sql
UPDATE users SET role = 'planning_director' WHERE id = 'ТУТ_ВАШ_UUID';
```

### 6. Заполнить тестовыми данными (Семо/Seed)
Чтобы вы могли сразу тестировать систему, выполните скрипт:
- `/supabase/seed.sql` в вашем SQL Editor. 
*Скрипт создаст 8 уникальных машин, десятки производственных этапов, позиций снабжения и несколько инвойсов.*

### 7. Запустить сервер
```bash
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000), войдите в систему и пользуйтесь!

## Роли доступа в системе (RBAC)
Система работает с 8 жестко заданными ролями, защищенными RLS-политиками на уровне БД:
1. **Директор по планированию** (Администрирование, Users)
2. **Финансовый директор** (Инвойсы)
3. **Начальник снабжения** (Доступ к /supply)
4. **Менеджер по продажам** (Sales Plan)
5. **Начальник ПДО**
6. **Начальник производства** (Таблицы цеха, Гант-график, сдача осей)
7. **Мастер** (Не может редактировать сроки, только статусы работ)
8. **Технолог** (Может указывать номенклатуру снабжения, но не назначать цены)

## Модули системы
- **Дашборд:** Стартовая страница с метриками и последними уведомлениями.
- **Sales Plan:** Управление портфелем проектов завода.
- **Производство:** Оперативное отслеживание по цехам.
- **Гант-график:** Визуальная раскладка сроков выполнения (планус vs. факт).
- **Снабжение:** Обособленная зона закупки номенклатуры и контроля бюджетирования.
- **Инвойсы:** Автоматизированный финансовый трекинг и выставление счетов.
- **Уведомления:** Глобальная система оповещений (просрочки, отгрузки, события) с Live-индикатором.
